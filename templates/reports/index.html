FAQ: Пролонгация

Что попадает в экспорт: активные полисы с датой окончания страхования в выбранном диапазоне (date_from — date_to).

Как формируется: данные сортируются по филиалу, дате окончания и номеру полиса. В отчете 26 столбцов: базовые поля заполняются автоматически, остальные предназначены для ручного заполнения.

Почему полис может не попасть:

  • дата окончания вне выбранного периода;
  • у полиса статус «Полис активен» = нет;
  • задан дополнительный фильтр по филиалу branch;
  • ошибка в диапазоне дат (дата начала позже даты окончания).
FAQ: Очередные взносы

Что попадает в экспорт: только неоплаченные платежи (paid_date пусто) по активным полисам в выбранном периоде по дате платежа (due_date), кроме первого платежа первого года (1.1).

Что рассчитывается автоматически:

  • статус рассрочки: «годовой» или «рассрочка»;
  • «Этот взнос» в формате N из M;
  • «Крайняя дата запроса счета» = дата платежа минус 2 недели.

Почему платеж может не попасть:

  • платеж уже оплачен;
  • полис неактивен;
  • это первый платеж первого года (1.1);
  • дата платежа вне выбранного диапазона;
  • задан фильтр по филиалу branch.
FAQ: Четверговый отчет

Что попадает в отчет:

  • полисы с признаком policy_uploaded = False;
  • неоплаченные платежи (paid_date пусто) по активным полисам.

Как работает дата «для раздела 2»: учитываются только платежи с договорной датой due_date <= payment_date. Если дата не указана, берется текущая дата.

Почему в строке может быть две причины: для одного полиса одновременно найдены «не подгружены документы» и «нет данных об оплате»; причины объединяются в одной строке.

Новый столбец «Участие брокера»: показывает значение из карточки полиса (Да / Нет).

Почему запись может не попасть: полис подгружен и нет неоплаченных платежей, подходящих под выбранную дату.

Подсветка для приоритизации: строки, где Участие брокера = Нет, выделяются мягким фоном. Это визуальный приоритет, на критерии отбора в отчет не влияет.

FAQ: Отчет по КВ

Что попадает в отчет: платежи только выбранной страховой компании, где есть фактическая дата оплаты (paid_date заполнена), но еще нет даты согласования со страховой (insurer_date пусто), и КВ в рублях больше нуля.

Как формируется: выгрузка создается в Excel, данные сортируются по дате оплаты и номеру полиса. В отчете 12 столбцов: полис, страхователь, даты страхования, страховая сумма, премия, КВ %, КВ руб и филиал.

Почему запись может не попасть:

  • платеж еще не оплачен;
  • акт уже согласован со страховой компанией;
  • КВ в рублях равно нулю;
  • платеж относится к другой страховой компании.
FAQ: Отчет по 3%

Что попадает в отчет: платежи за выбранный квартал по дате согласования акта с СК (insurer_date), где дата согласования заполнена.

Что формируется в выгрузке: Excel на 14 столбцов с данными по ДФА, полису, клиентам, страховой премии, вознаграждению 3%, датам оплаты и согласования, страховщику, филиалу, КВ и Инфо 3.

Почему платеж может не попасть:

  • дата согласования акта с СК не заполнена;
  • дата согласования акта с СК вне выбранного квартала;
  • по выбранному кварталу нет согласованных платежей.
FAQ: КВ за месяц

Что попадает в отчет: платежи портфеля за выбранные месяц и год по дате согласования акта с СК (insurer_date), где КВ в рублях больше нуля.

Что формируется в выгрузке: Excel на 9 столбцов — Страховщик, Номер полиса, Номер ДФА, Страхователь, КВ в %, Страховая премия, Дата фактической оплаты, КВ (в руб), Филиал.

Как считается КВ в %: фактический процент рассчитывается из соотношения КВ (в руб) / Страховая премия * 100.

Почему запись может не попасть:

  • дата согласования акта с СК вне выбранного месяца/года;
  • дата согласования акта с СК не заполнена;
  • КВ в рублях равно нулю;
  • по выбранному периоду нет оплаченных платежей.
FAQ: Экспорт CSV

Что попадает в экспорт: платежи за выбранный период по договорной дате платежа (due_date между датой начала и датой окончания включительно).

Что генерируется: CSV-файл в кодировке UTF-8 (с BOM) с разделителем ;. В выгрузке поля по ДФА, клиенту, страховщику, страхователю, суммам, КВ, виду страхования, филиалу, месяцу и участию брокера.

Почему запись может не попасть:

  • договорная дата платежа вне указанного диапазона;
  • ошибка в периоде (дата начала позже даты окончания);
  • не заполнены обязательные даты периода;
  • по выбранному диапазону нет платежей.
