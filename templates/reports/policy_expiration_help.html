FAQ: Экспорт «Пролонгация»

Назад к экспортам
Кратко (TL;DR)
  • Выгружаются полисы, у которых дата окончания страхования попадает в выбранный диапазон.
  • Выгрузка включает 26 столбцов: 7 автозаполняются системой, остальные подготовлены для ручной работы.
  • Файл группируется по филиалам и содержит служебную строку «кто заполняет какие колонки».
Параметры запуска
Параметр Обязательность Как влияет
date_from, date_to Обязательно Фильтр по дате окончания страхования: end_date >= date_from и end_date <= date_to.
branch (технический) Опционально Если передан в URL, ограничивает выгрузку одним филиалом.
Проверка диапазона Обязательно Если date_from > date_to, экспорт не формируется.
Как формируется файл
  1. Система отбирает полисы по диапазону даты окончания страхования.
  2. Полисы сортируются по филиалу, затем по дате окончания и номеру полиса.
  3. В Excel создаются: заголовок отчета, строка заголовков, строка распределения ответственности и блоки филиалов.
  4. Для каждого полиса автоматически заполняются базовые столбцы: номер полиса/ДФА, лизингополучатель, страховщик, страхователь, дата окончания и объект.
  5. Колонки для продления (новый страховщик, сумма, условия и т.д.) оставляются пустыми для ручного заполнения.
  6. В «Примечания» автоматически добавляются маркеры:
    • «без брокера»
    • «ДФА закрыт»
Особенности отображения
  • Если ДФА закрыт, строка подсвечивается желтым (приоритет над другими подсветками).
  • Если брокер не участвует, строка подсвечивается серым.
  • Для строк данных используется шрифт Times New Roman 9 pt.
  • Границы, объединения и переносы настроены так, чтобы файл был удобен для дальнейшей ручной доработки.
